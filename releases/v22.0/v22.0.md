# v22.0 — 2026-02-25

Six weeks of development. Major work across the desktop, the specialist system, email, and infrastructure.

---

## Desktop & UI

The Desktop received a comprehensive overhaul across eight areas: the ChatInput area was redesigned as a unified container with attachment chips inside; the 2164-line `ContextMenu.tsx` monolith was decomposed into 16 focused files with consistent ordering across all menus; desktop icons are now configurable (three size presets, grid flow, alignment, sort order, extensions toggle); editor toolbars were removed from individual editors and unified into a context-aware PathBar with macOS-style breadcrumbs; error components were rebuilt with CSS variables throughout; initial load performance improved significantly (DOM nodes reduced 71%, Monaco and xlsx now lazy-loaded); rubber band marquee selection was added to the desktop, and `@dnd-kit` was replaced with native HTML5 drag; window open/close animations were added, QuickLook now animates from its source icon, and active/inactive windows have differentiated shadows.

## ClaudePanel

Typing `@` in the chat input opens a searchable file picker for attaching Desktop files. The send flow was fixed for a race condition that dropped messages on session transitions. Tabs now sort interactive specialists first, followed by background agents, then archived sessions.

## Specialist System

Subagents render live transcripts inside their spawn cards as they run — tool calls, text output, and thinking blocks visible in real time. `AskUserQuestion` now renders as an inline card in the transcript with selectable options and text input, replacing the terminal prompt. When Chief spawns an interactive specialist with a spec, the spec is injected automatically at session start. Skill invocations render as compact labeled chips instead of raw YAML frontmatter. Five new specialist subagents added: `data-scientist`, `best-practices`, `practitioner`, `skeptic`, `ux-perspective`.

## Apps

**Guide** — New fullscreen app in the Dock. Visual editorial guide to Claude OS with 20 chapters across 8 parts, scroll progress tracking, and chapter navigation menu.

**Projects** — Rebuilt as a grouped card grid with filter bar, detail slide-over panel, GROUP.md support for group-as-entity, and interview value badges.

**Contacts** — Rebuilt from a phone book into a relationship intelligence feed. Reverse-chronological activity stream, TodayPeopleStrip showing contacts active today, lazy search. Requires database migration (018).

**Observatory** — New Subagents tab with bar chart and detail table. `analytics("subagents")` MCP operation added.

## Email

Classifier pipeline now loads rules at startup and caches them in memory for O(1) lookup. Deduplication check prevents double-classification on reruns. Three new MCP operations: `email("rules")`, `email("rule_create")`, `email("rule_delete")`. Classifier prompt moved into the engine module. Email Intel widget redesigned with full subject lines and color-coded badges. `current_datetime` now injected into classifier context; HTML escaping fixed for draft content.

## Memory & Handoffs

Handoff documents now include two new sections: **Immediate Re-read List** (files to read first) and **Active Skills** (skills in progress). Summarizer prompt enforces Edit tool bookends, adds verify-retry-fallback chain, and injects previous handoffs for cumulative context.

## Infrastructure

**tmux injection** — Third fix for the Ink/React render race. Content-to-Enter delay raised to 300ms, per-pane threading locks prevent interleaved injections, Telegram service uses async injection to avoid blocking the event loop.

**Backend async** — Sync I/O in the email pipeline and projects API offloaded to `asyncio.to_thread()`, unblocking the FastAPI event loop during Apple Mail AppleScript calls and filesystem scans.

**Absolute paths** — File paths throughout the Dashboard are now absolute. `pathUtils.ts` centralizes path utilities. `fetchFileTree()` returns `{ repoRoot, desktopRoot, tree }`.

**team() cleanup** — `reply` and `subscribe` operations removed. `message` now includes sender's `conversation_id` so recipients can reply directly.

**CLAUDE.md & role files** — File discipline rules added (specs die on ship, reports filed to logs/). Email triage handling rules added to Chief. Bootstrap Instructions sections added to Chief and Project roles. Curator gains a Spec Cleanup phase in memory consolidation.
